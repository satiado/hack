- Aluno: 
    Chega em frente do painel, faz login com face.id com dados do conecta, caso não seja possível, ele faz login com teclado digital da tela com os dados cadastrais do conecta, e então a porta abre.
    Entra na biblioteca, a porta fecha, escolhe (ou não) um livro.
    Na saída, possuí opção de registrar como ter pego um livro ou não: 
        Se escolhido o livro, o aluno passa o livro no sensor RFID e faz novamente o login para sair da sala e os dados serem relacionados.
        Se não, o aluno sai da sala sem livro, apenas realizando o login facial para registrar a saída.
    Solicita agendamento das salas.
    Possuí prazo limite para devolução

- Professor: 
    Entra e sai da biblioteca com Face.id ou login no teclado digital.
    Realiza empréstimo de livros na saída.
    Solicita agendamento das salas.
    visualiza relatórios.

- Vigiante:
    Possuí chave para porta para abri-la em casos de queda de energia.
    Recebe notificação de entrada na biblioteca.
    Tem resposabilidade de supervisionar biblioteca.
    Ao final do expediente, deve realizar varredura dos livros, checando se todos livros estão correspondentes aos dados do banco de dados, onde se houverem faltas não registradas, deve-se realizar verificação em câmeras de segurança.

- Livros:
    Possue cadastro utilizando RFID.
    Possue status de disponibilidade.

- Administrador: 
    Cadastrar livros.
    Gerar relatórios.
    Visualizar todos livros em disponibilidade ou emprestados.
    Visualizar o tempo que aluno está na sala.
    Recebe alertas de atraso.

    detecta rostos em tempo real
    compara com modelos faciais
    lê etiquetas RFID dos livros
    associa livro ao aluno
    calcula prazos automaticamente
    detecta livros desaparecidos
    verifica conflito de agenda
    gera alertas automáticos
    atualiza painel em tempo real

- ESP32:
    Lê dados do sensor RFID, se houverem dados: manda para banco de dados e relaciona o livro que foi pego e o aluno que pegou.
    O aluno que pegou é relacionado quando na saída escolhe registrar a retirada de um livro, então pede o código para o sensor RFID que faz com que o ESP32 mande qual o livro que esta sendo emprestado e então relacione com o aluno que fez login para realizar a saída da biblioteca

- Aluno quer entrar na biblioteca, scan facial processa se aluno está em banco de dados, se estiver, um esp32 envia um comando que permite a passagem pela trava eletrônica.
Lá dentro o aluno escolhe um livro, este livro já deve estar posteriormente cadastrado com RFID no banco de dados. Na saída, o aluno escolhe se ele quer sair sem o livro ou empresta-lo por determinado tempo. Se o aluno escoher fazer o empréstimo, então o sistema verifica o RFID na saída, e scaneia o rosto novamente, cruzando os dados entre o aluno que sai e o livro que é verificado.
Para o devido controle, um vigilante possuí dados de estrada e saída de cada aluno: de quando ele entrou, de quando ele saiu, de quanto ele ficou na sala, se ele pegou um livro ou não, e se houve movimentação nas prateleiras que não condiz com o dado de retirada de livro, movimentação que é verificada por meio de uma câmera com visão computacional apontada para a estante.
O esp recebe a informação de se o sensor está identificando um livro e qual livro é, então manda o dado para o wifi e comunica com o sistema localizado no computador do administrador, para assim cruzar os dados de quem pega os livros e que livros são pegos.
